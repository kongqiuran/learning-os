# Learning OS V2 上线运维与收费操作

## 1. 服务结构

生产环境包含 `backend`、`worker`、`frontend` 三个常驻服务。API 只创建生成任务，`worker` 从 SQLite 领取任务并执行。Worker 每 15 秒写入心跳；任务心跳超过 90 秒会在重启后恢复，最多执行两轮，最终失败自动返还额度。

## 2. 健康检查

- `/api/health`：兼容探针。
- `/api/health/live`：API 进程存活。
- `/api/health/ready`：数据库、上传目录、Worker 心跳和磁盘空间。
- `/api/health/metrics`：生成任务状态数量，不返回用户资料。

## 3. 对象存储备份

在服务器 `.env.production` 配置 `BACKUP_S3_ENDPOINT`、`BACKUP_S3_BUCKET`、`AWS_ACCESS_KEY_ID`、`AWS_SECRET_ACCESS_KEY` 和区域。凭据不得提交到 Git。

手动验证一次备份：

```bash
cd ~/learning-os
docker compose --profile ops run --rm backup
```

确认成功后，为服务器用户配置每天低峰期执行的定时任务：

```bash
cd ~/learning-os
(crontab -l 2>/dev/null; echo '20 3 * * * cd /home/learning/learning-os && docker compose --profile ops run --rm backup >> /home/learning/learning-os/backup.log 2>&1') | crontab -
```

备份保留最近30个每日版本和12个每月版本，并在对象元数据中保存 SHA-256。恢复工具只恢复到指定临时目录，不会自动覆盖生产数据：

```bash
cd ~/learning-os
docker compose --profile ops run --rm backend python scripts/restore_backup.py --key '<对象键>' --output /data/restore-check
```

完整性检查通过后仍需人工停止服务、再次备份当前数据并确认替换。

## 4. 99元单课人工激活

付款前必须让用户确认课程、学期结束日、三类整理各3次、课程问答100次、失败返还、不自动续费及人工退款方式。

收到付款后执行：

```bash
cd ~/learning-os
docker compose exec backend python scripts/activate_course_entitlement.py --email '<用户邮箱>' --course-id <课程ID> --semester-end <YYYY-MM-DD> --payment-reference '<付款参考号>'
```

激活后让用户打开用户中心，核对课程名称、到期日和四项剩余额度。操作记录不得删除；撤销或延期应保留历史依据。

## 5. 部署与恢复验证

数据库迁移前先完成对象存储备份。部署后检查三个常驻服务、readiness、Worker 日志以及前端三场景页面。至少每月将一个备份恢复到临时目录并执行完整性检查。
