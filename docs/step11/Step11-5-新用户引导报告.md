# Learning OS Step11-5 新用户引导报告

## 1. 实现结果

新用户在没有课程时会看到非阻塞欢迎卡片，明确说明三步路径：

1. 上传课程资料；
2. AI 整理知识；
3. 开始学习。

欢迎卡提供“创建第一门课程”主动作和关闭按钮。关闭状态按 user_id 保存在浏览器本地，不会影响其他账号；已有课程的老用户不显示该引导。

课程流程的空状态同时增强：

- 没有课程：继续保留“创建你的第一个学习空间”和创建按钮；
- 没有资料：显示四类上传入口，并建议先上传教材或课件；
- 有资料但没有学习内容：学习内容区域直接显示“开始整理课程内容”按钮，不再要求用户寻找左侧操作。

## 2. 修改文件

- `frontend/src/components/domain/WelcomeGuide.tsx`
  - 欢迎文案、三步路径、关闭状态和创建课程动作。
- `frontend/src/pages/DashboardPage.tsx`
  - 只在 course_count 为 0 时接入引导。
- `frontend/src/components/course/LearningPackageView.tsx`
  - 根据是否已有资料展示可执行的学习内容空状态。
- `frontend/src/pages/CourseSpacePage.tsx`
  - 复用原生成 mutation，向导航和空状态提供同一个生成动作。
- `tests/test_onboarding_frontend.py`
  - 检查新老用户条件、非阻塞关闭、三步内容和直接生成入口。

## 3. 行为边界

- 不新增后端字段，不修改 User 模型；
- 不修改登录、Session、隐私授权或生成状态机；
- 不自动创建课程、不自动上传、不自动触发付费模型调用；
- 引导关闭不影响核心功能；
- 已有课程用户不显示欢迎卡；
- 同一浏览器不同账号使用独立关闭 key。

## 4. 测试范围

- 新账号空 Dashboard 显示欢迎引导；
- 引导包含三个指定步骤；
- 引导可以关闭且不是阻塞弹窗；
- 已有课程账号不显示引导；
- 无资料课程引导分类上传；
- 有资料但未生成时提供直接生成按钮；
- 原 CourseNavigation 和 LearningPackage 生成逻辑保持一致。

## 5. 人工验收步骤

1. 新注册账号并完成隐私授权；
2. 确认 Dashboard 显示欢迎卡和三步说明；
3. 关闭后刷新，确认不再次显示；
4. 新建课程，确认欢迎卡自然消失；
5. 在空课程选择“上传教材”；
6. 上传成功后在学习内容空状态点击“开始整理课程内容”；
7. 确认生成中与完成后的现有状态流没有变化。
